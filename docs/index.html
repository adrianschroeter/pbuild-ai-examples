What is pbuild-ai?

pbuild-ai is an AI agent for the openSUSE pbuild tool. It automates package analysis, build fixing, upstream updates, and package generation using AI models provided by an openai api (e.g. ollama).

This site showcases practical examples demonstrating pbuild-ai's capabilities. Each example includes the source package, command used, expected results, and benchmark data. The examples also serve as a functional test suite for pbuild-ai.

pbuild-ai is designed to be used on git-based packages or entire projects. However, it can also be used on legacy package sources of OBS. Working on OBS projects requires some workarounds.

Extendable via skills and scripts. Global, project or package specific.

Automated package analysis
Intelligent build fixing
Upstream version updates
Vibe packaging via prompts
Secure by default

What is pbuild-ai NOT?

Fast: pbuild-ai often requires many steps and the speed depends heavily on the AI you use. It is actually horribly slow at the moment; its main value is that it can run fully automatically. However, we will improve the speed in the future.

At risk of high costs: We depend on an OpenAI API provider. Keep it under your control to manage costs. Alternatively, a system like an AMD Strix Halo only consumes 120 Watts of electrical power, making it environmentally friendly.

A tool for creating denial of service: pbuild-ai will not push changes or create pull requests by default. However, you could extend it for your project through custom tool scripts. Please use this capability responsibly.

Complete: We only have 10 skills at the moment. Tuning and further skills are needed.

Featured Examples

Practical demonstrations of pbuild-ai capabilities

Basic Package Analysis

Analyze a package source without making any modifications. Perfect for understanding package structure and dependencies.

pbuild-ai
basic analysis

Fix Build Failures

Automatically detect and fix build failures. The AI analyzes build logs and applies appropriate patches.

pbuild-ai --fix
fix intermediate

Update to Upstream

Update package to the latest upstream version, rebuild, and fix any compatibility issues automatically.

pbuild-ai --update
update advanced

Update Sources Only

Update package sources to the latest upstream version without running a test build.

pbuild-ai --update-only
update basic

Backport Security Fix

Apply a security fix from upstream using AI-guided patching with custom prompt and --fix mode.

pbuild-ai --prompt "Apply security fix from URL" --fix
security fix intermediate

Simple Modification

Make quick, targeted changes using the --modify flag with a simple text prompt.

pbuild-ai --modify "Add a systemd service file"
basic modify

Deep Analysis Mode

Interactive investigation session inside the build environment. Diagnose complex build failures step-by-step.

pbuild-ai --deep-analyze --interactive --fix
advanced debugging

Generate New Package

Create a complete RPM package from scratch based on a description. AI researches upstream and generates spec files.

pbuild-ai --generate "package ripgrep CLI tool"
generate advanced

Update Package with Own Tools

Update a package using custom tool scripts to extend pbuild-ai with package specific automation. Can also be used to gain more permissions, eg. for pushing generated content.

pbuild-ai --update --allow-tool-scripts
update custom advanced

Project Examples

Working with entire projects containing multiple packages

Test Build and Fix Project

Test build an entire project containing multiple packages and automatically fix any build failures across all packages.

pbuild-ai --fix
fix project

Update Sources Only

Update all package sources in a project to their latest upstream versions without running test builds. Fast update for preparation work.

pbuild-ai --update-only
update fast

Update, Build and Fix All

Complete project update workflow: update all sources to the latest upstream, test build everything, and automatically fix any compatibility issues.

pbuild-ai --update
update fix advanced
